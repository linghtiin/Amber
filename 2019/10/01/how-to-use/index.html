how to use hexo

发表于 2019-10-01 更新于 2019-12-08 分类于 Blog ， hexo

简介

在Hexo中支持Markdown语法，但有部分语法会导致不可预测的Bug。
所以官方推荐以数据块的形式替代部分Markdown原生语法。

框架

命令

  • hexo
    • init [folder]
      文件夹初始化
    • new [layout] <title>
      • -p,–path
        指定文章路径
      • -r,–replace
        替换同名文章
      • -s,–slug
        使用文章Slug,作为文件名与URL
    • publish [layout] <filename>

      发布草稿

    • server

      启动服务器

      • -p,–port
        重设端口
      • -s,–static
        只使用静态文件
      • -l,–log
        启动日志记录
    • generate

      编译文件
      该命令可被简写为hexo g

      • -d,–deploy
        立即部署
      • -w,–watch
        监视文件变动
      • -b,–bail
        生成过程中如果发生任何未处理的异常则抛出异常
      • -f,–force
        强制重新生成
      • -c,-concurrency
        设定最大生成文件数量
    • deploy

      发布文件
      该命令可被简写为hexo d

      • -g,–generate
    • render

      渲染文件

      • -o,–output
    • migrate

      从其他博客系统中迁移内容

    • clean

      清除缓存文件

    • list

      列出网站资料

    • version

      显示hexo版本

文件夹结构

  • project
    • .deploy_git
      发布用版本管理库
    • node_modules
      环境包文件夹
    • public
      存放编译后网页文件的文件夹
    • scaffolds
      存放文件模板的文件夹
    • source
      存放源文本的文件夹
      • posts
        发布文章的文件夹
      • drafts
        草稿文章的文件夹
    • themes
      存放主题的文件夹
